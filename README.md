# ADC 센서 데이터 보정을 위한 이동 평균 필터 설계 및 시뮬레이션

C언어로 구현한 이동 평균 필터 시뮬레이터입니다.

노이즈가 섞인 ADC 센서값을 소프트웨어 필터로 보정하고, 결과를 파일로 저장합니다.

---

## 배경 지식

### ADC(Analog to Digital Converter)란?

온도계, 압력계처럼 실제 세상의 신호는 연속적인 아날로그 값입니다.

컴퓨터는 0과 1만 처리할 수 있기 때문에, 이 아날로그 값을 숫자로 변환해주는 장치가 **ADC**입니다.

```
온도 센서 (25.3℃)  →  [ADC 변환]  →  숫자 512  →  컴퓨터 처리
```

이 프로젝트는 **10비트 ADC**를 기준으로 합니다.

- 표현 범위: 0 ~ 1023
- 중간값: 512

### 왜 필터가 필요한가?

센서는 항상 정확한 값만 내보내지 않습니다.

전기적 간섭, 전원 잡음, 온도 변화 등으로 인해 실제 값 주변에서 **불규칙하게 떨리는 현상**이 생깁니다.

이를 **노이즈(Noise)** 라고 합니다.

```
이상적인 센서:  512  512  512  512  512  ...  (흔들림 없음)
실제 센서 출력: 524  536  505  494  532  ...  (불규칙하게 튐)
```

하드웨어를 바꾸지 않고, **C언어 코드만으로** 이 노이즈를 걸러내는 필터를 구현합니다.

### 이동 평균 필터(Moving Average Filter)란?

가장 최근 N개의 값을 평균 내는 방식입니다.

노이즈는 무작위로 높거나 낮게 튀기 때문에, 여러 값을 평균 내면 서로 상쇄되어 원래 신호에 가까워집니다.

```
입력값:  524  536  505  494  532  510  487  522
                     ↓ 8개 평균
필터값:  (524+536+505+494+532+510+487+522) ÷ 8 = 513.75
```

실제 신호인 512에 훨씬 가까워진 것을 확인할 수 있습니다.

## 파일 구성

```
adc_filter/
├── adc_filter.h    ← 설계도: 구조체 정의, 상수, 함수 목록
├── adc_filter.c    ← 핵심 엔진: 필터, 비트 연산, 통계, 파일 저장
└── main.c          ← 진입점: 난수 생성, 루프 제어, 결과 출력
```

---

## 실행 방법

```bash
# 1단계: 컴파일
gcc -Wall -Wextra -o adc_filter main.c adc_filter.c -lm

# 2단계: 실행
./adc_filter
```

실행 후 현재 디렉터리에 `adc_report.txt`와 `adc_data.csv`가 자동 생성됩니다.
